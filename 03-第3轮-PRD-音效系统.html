第 3 轮 · PRD · 音效系统

让游戏「有声」—— BGM 自备文件循环播放 + SFX 全部 Web Audio API 代码合成，零下载零依赖

课程版本：B2 · 第 3 轮 / 共 6 轮 · 文档版本：v1

本轮目标（必读） 第 1 轮已把设置面板的「BGM 音量」和「SFX 音量」两个滑块做好，状态写入 localStorage，本轮只是把音频接上线——让这两个滑块真正驱动声音。本轮不重做任何 UI，不新增任何设置项。粒子升级（通关撒金币、爆分粒子特效）同属原计划第 3 轮，但不在本 PRD 范围内，留给后续轮次单独处理。

一、产品概述

本轮目标：为小丑牌游戏接入完整音效体系，让玩家在出牌、计分、通关、商店等全流程中都有听觉反馈，大幅提升游戏手感与沉浸感。

音效分两类：

  • BGM（背景音乐）：由学员自备音频文件放入 public/ 目录，游戏运行时循环播放，音量受设置面板「BGM 音量」滑块实时控制。
  • SFX（操作音效）：全部用浏览器内置的 Web Audio API 代码合成，振荡器 + 包络生成像素电子音，零音频文件、零外部依赖、零下载、不联网。音量受设置面板「SFX 音量」滑块实时控制。

二、功能模块

M1 · 音频系统架构与 Autoplay 解锁策略

1.1 Autoplay 解锁约束 锁定

浏览器 Autoplay 策略：页面加载后不得自动播放任何音频 现代浏览器（Chrome / Firefox / Safari）在用户首次与页面产生交互之前，禁止脚本发起音频播放，强行播放会被静默拦截或抛出 NotAllowedError。

产品行为要求：

  • 页面加载完成后，不自动初始化 AudioContext，不自动播放 BGM，不报错。
  • 首次用户交互（点击任意按钮或手牌）时，才初始化 AudioContext 并解锁 BGM 播放。「首次交互」指用户在当前页面生命周期内第一次触发 click 事件。
  • 解锁后，BGM 立即开始循环播放（若音量不为 0）；此后所有 SFX 调用均可正常发声。
  • 解锁逻辑只触发一次，后续交互不重复执行。

1.2 整体架构约束 锁定

项要求
音频技术BGM 用 HTML <audio> 元素或 AudioContext + AudioBufferSourceNode 加载文件；SFX 全部用 Web Audio API OscillatorNode + GainNode 合成
外部依赖零外部音频库（不引入 Howler.js、Tone.js 等任何第三方音频库）
网络请求SFX 不产生任何网络请求；BGM 仅加载本地 public/ 目录下的文件，不联网
API Key不需要任何 API Key

M2 · BGM（背景音乐）

2.1 文件约定 锁定

项规则
文件位置public/bgm.mp3（首选）；也可放 public/bgm.ogg 或 public/bgm.wav，工程师按顺序尝试加载，取第一个存在的文件
文件来源由学员自备，课程不提供版权音乐。PRD 只规定文件名约定，不规定音乐内容。
文件缺失降级文件不存在时，BGM 模块静默降级，不报错、不弹框、不影响游戏运行。SFX 不受影响，正常发声。

2.2 播放行为 锁定

  • BGM 在「首次用户交互解锁」后立即开始播放（若音量不为 0）。
  • 循环播放，不自动结束。
  • BGM 贯穿所有游戏状态（playing / shop / won / lost），不因状态切换而中断或重播。
  • 音量由设置面板「BGM 音量」滑块（0–100）实时控制，映射关系见 M4。
  • BGM 音量为 0 时，BGM 静音（暂停播放或将音量设为 0，均可接受）；音量从 0 调回正值时，BGM 恢复播放。

M3 · SFX（操作音效）—— 全代码合成

合成原则 所有 SFX 均通过 Web Audio API 的振荡器（OscillatorNode）+ 增益包络（GainNode）实时生成，无需任何音频文件。音色描述仅作为方向性指引，具体频率、波形、ADSR 参数由工程师在实现时决定，只要听感符合描述即为达标。

M4 · 音量控制与设置接线

4.1 音量接线 锁定

本轮核心工作：让第 1 轮已有的滑块真正驱动声音 settings.bgmVolume（默认 50）和 settings.sfxVolume（默认 70）在第 1 轮已存在于 App.vue，由 watch(settings, ...) 持久化到 localStorage key = balatro.settings。本轮在此基础上接线，不修改设置面板 UI，不新增 ref，只让这两个值驱动音频。
滑块对应 key范围默认值 锁定控制范围
BGM 音量settings.bgmVolume0–10050BGM 播放音量（HTML audio 的 volume 属性，映射：滑块值 ÷ 100 = 0.0–1.0）
SFX 音量settings.sfxVolume0–10070所有 SFX-01 至 SFX-17 的 GainNode 增益（映射同上）

4.2 实时响应要求 锁定

  • 拖动 BGM 音量滑块时，BGM 播放音量即时响应（不需要松开鼠标才生效）。
  • 拖动 SFX 音量滑块时，下一个 SFX 触发时以新音量播放（已在播放中的 SFX 不受影响，自然结束即可）。
  • BGM 音量调至 0：BGM 停止发声（静音或暂停均可接受，暂停更省资源）。
  • SFX 音量调至 0：所有 SFX 静音（播放时 GainNode 增益设为 0）。
  • 刷新页面后，localStorage 中保存的音量值被读回，BGM 以此音量继续播放（待首次交互解锁后），SFX 以此音量发声。

4.3 动画速度倍率与音效的关系 锁定

音效不随动画速度变调变速。设置面板中「动画速度」（慢 / 普通 / 快）只影响动画时间线的时间缩放（第 1 轮 PRD §5.1），不影响任何 SFX 的音高、时长或触发时机。每个音效在其对应的真实事件点触发，与动画时长无关。

三、关键交互说明

3.1 Autoplay 解锁流程

页面加载 → 音频未初始化，静默等待 → 用户首次点击（任意按钮/牌） → 初始化 AudioContext → BGM 开始播放（若音量 > 0 且文件存在） → 后续 SFX 正常发声

3.2 BGM 文件缺失降级

若 public/bgm.mp3（及其他备选格式）均不存在，系统行为：

  • 控制台可以有一条 console.warn 提示（非必须），但绝对不弹出任何用户可见的错误提示。
  • 游戏正常运行，BGM 模块处于「无文件」状态，BGM 音量滑块拖动不报错。
  • SFX 完全不受影响，正常发声。

3.3 SFX 并发播放

同一时刻可能有多个 SFX 同时触发（例如：多个 Joker 同时触发，或发牌时多张牌错峰播放），系统必须支持 SFX 并发不相互阻断。Web Audio API 天然支持多个 OscillatorNode 同时运行，工程师无需做串行队列。

3.4 出牌全流程音效时序

以下为出牌一手时，SFX 与 handlePlay() 动画步骤的对应关系（仅示意时序，不规定具体 ms 数，与动画速度倍率解耦）：

七、验收清单

BGM 基础行为

  • 页面加载后，BGM 不自动播放，不报错，不弹窗
  • 首次点击任意按钮或手牌后，BGM 开始循环播放（前提：音量 > 0 且文件存在）
  • BGM 贯穿 playing / shop / won / lost 所有状态，状态切换不中断播放
  • 拖动「BGM 音量」滑块，播放音量即时响应变化
  • BGM 音量调至 0，BGM 停止发声
  • BGM 音量从 0 调回正值，BGM 恢复播放
  • 刷新页面后，BGM 音量设置保持（从 localStorage 读回），首次交互后以保存的音量播放

BGM 降级

  • public/bgm.mp3（及其他备选格式）不存在时，页面正常加载，无用户可见报错
  • 文件缺失时，BGM 滑块可以正常拖动，不报错
  • 文件缺失时，所有 SFX 正常发声，不受影响

SFX 出牌全流程

  • 点击手牌选中：发出 SFX-01（短促上扬单音）
  • 点击已选中手牌取消：发出 SFX-02（下行收回音）
  • 点击出牌按钮，飞牌动画开始：发出 SFX-03（嗖声）
  • 逐张高亮计分时，每张各发出 SFX-04 一声，5 张音高依次递增，听感有明显上行感
  • Joker 触发金光时，每个触发的 Joker 各发出 SFX-05 一声
  • 公式大字爆出时，发出 SFX-06（爽感冲击音，全局最响）
  • 补牌发牌时，每张入手各发出 SFX-07 一声，随错峰动画依次触发

SFX 胜负与关卡

  • 3 关全过，进入 won 状态时，发出 SFX-08（胜利琶音，可感知到庆典感）
  • 某关失败，进入 lost 状态时，发出 SFX-09（失败下行音，与 SFX-08 音色明显区分）
  • 小/中盲注通关进入商店时，发出 SFX-10（进店叮咚音）

SFX 商店与按钮

  • 商店中点击「购买 $X」成功买卡时，发出 SFX-12（愉悦双音叮当）
  • 点击弃牌按钮时，发出 SFX-14（低沉短促噗音）
  • 点击其他普通按钮（跳过、重新开始、关闭设置、排序）时，发出 SFX-13（通用咔哒）
  • 拖动设置面板滑块时，SFX-15 节流触发（每 100ms 最多一次），不影响正常操作
  • 切换动画速度按钮时，发出 SFX-15 一声

SFX AI 出牌

  • 点击「🤖 AI 出牌」进入思考状态后，SFX-16（科技脉冲音）约每 400ms 触发一次，直至思考结束
  • AI 思考完成、即将出牌时，发出 SFX-17（确定叮音），紧接在最后一次 SFX-16 之后

SFX 音量控制

  • 拖动「SFX 音量」滑块，下一次触发的 SFX 以新音量发声
  • SFX 音量调至 0，所有 SFX 静音（触发但无声）
  • 刷新页面后，SFX 音量设置保持，后续操作以保存的音量发声

并发与稳定性

  • 多个 Joker 同时触发时，SFX-05 并发播放不互相阻断
  • 发牌补牌时，多个 SFX-07 错峰触发不互相阻断
  • 快速点击多张手牌时，SFX-01/02 不因并发而报错或静默
  • 构建通过：npm run build 结果 0 error、0 warning
